PacketReceiver

整个通信基于UDP, 所以不用考虑连接维护这种麻烦的工作.
还有就是使用的是侦听器的设计模式, 可以根据收到的UDP包的内容进行回调(后面详述), 因此可以很方便地进行二次开发(好吧没人会用这个东西的)

ChatCLI 负责翻译用户指令然后操作 ChatCore 发出消息.
ChatCore 封装各种事务比如上线和下线消息的发送.
PacketReceiver 负责侦听端口并根据包内容回调.
ContactList 也就是联系人表, 包含了名字到IP及端口的映射. 表的内容由 ChatCore 负责维护.

启动

  • ChatCLI 构造一个 ChatCore
  • ChatCore 读取配置
    • 并生成自己的名片
    • 初始化一个 ContactList, 然后将自己的名片加进去
    • 初始化一个PacketReceiver, 监听预定的端口.
    • 广播一个上线消息.

封装

消息首先会根据消息类型被转化为相应的 json , 然后以 utf-8 获得其编码, 最后通过 UDP 发送出去. 接收方根据同样方法重建消息.
可用的消息有4种: hello(握手), re-hello(响应握手), msg(消息), exit(下线)
在 PacketReceiver 和 ChatCore 中都封装了相应的包到达事件, 可以通过侦听器获得这些消息.

基本就是 
            接收端启动
发送端启动--->提示对方上线
接受<--------回送身份
发送消息----->接收并显示
发送端退出--->提示对方下线

已知的缺陷

  • 发送下线通知时只是单纯广播到与自己相同的端口, 而不是按地址表逐一发送下线消息.
